cloudpivot-ecc · Rules & Skills 使用说明

面向云枢 AI-Platform-v1.1 的协作配置包。用于把 Cursor / Claude Code / VSCode 的规则、 skills、hooks 和编辑器配置快速注入目标项目，并保持可升级。

1) 项目情况与定位

  • 技术栈基线：Vue 2.7 + TypeScript 4.1 + Vuex 3 + Vue Router 3。
  • 协作资产：Rules（约束）+ Skills（任务路由）+ Hooks（安全/提交保护）+ IDE 配置。
  • 工程规范：语法/安全/性能规则提炼自 ECC universal，并适配云枢 Vue2 monorepo。
  • 定制策略：默认通用，行业定制可参考 bankcomm 方案，建议走扩展点降低升级成本。

2) 快速使用

terminal — bash
# 安装
npm i -g cloudpivot-ecc --registry=https://nexus01.authine.cn/repository/npm-cloudpivot/

# 注入到项目（默认 cursor + claude-code + vscode）
cecc init /path/to/your/project

# 仅安装某类
cecc init /path/to/your/project --harness cursor

# 冲突策略（默认）
# - 同名文件：覆盖
# - 同名目录：增量合并

# 强制先删后装
cecc init /path/to/your/project --force

# 体检、升级、卸载
cecc doctor /path/to/your/project
cecc update /path/to/your/project
cecc uninstall /path/to/your/project

3) Rules 与 Skills 的作用

Rules（规则）

  • 限制技术栈与改动边界（避免误改产品内核）。
  • 约束安全、i18n、Git 流程、包依赖、性能策略。
  • 共 9 条 .mdc，见下方完整清单。

Skills（技能）

  • 按任务意图路由到具体实践文档。
  • 覆盖 API、组件、门户、表单、二开、安全、review 等场景。
  • 共 17 个 skill，见下方完整清单。

Harness（安装包）

  • 按 IDE 分发：Cursor / Claude Code / VSCode。
  • 通过 cecc init --harness 选择安装范围。
  • 共 3 种 harness，见下方完整清单。

Hooks（自动化）

  • 敏感文件拦截、secret 检测、格式化、提交审计。
  • 配置在 hooks.json，脚本从 ECC 拉取。
  • 共 15 个事件，见下方完整清单。

9) 常用操作指令

terminal — bash
# 列出可安装项
cecc list

# 干跑（只看将执行操作）
cecc init . --dry-run

# 带 hooks 安装
cecc init . --with-hooks --ecc-path ~/gitlab/ECC

# 仅恢复备份场景卸载
cecc uninstall . --restore
